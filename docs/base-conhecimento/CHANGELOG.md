## Convenção de versionamento

Cada documento da Base de Conhecimento segue versionamento semântico simplificado:

- **vX.0** — versão major: reescrita estrutural ou mudança de modelo conceitual
- **vX.Y** — versão minor: ajuste de conteúdo, adição de seção, correção de dados
- **vX.Y.Z** — patch: correção de erro factual, typo, ajuste cosmético

Toda alteração em documento gera entrada neste CHANGELOG.

## Convenção de atualização de documento

Quando atualizar qualquer documento da Base:

1. **Verificar consistência cruzada** — uma mudança em Doc 03 (preço) pode impactar Doc 06 (ROI), Doc 08 (proposta), Doc 09 (margem), Doc 10 (escopo) e Doc 11 (renovação). Verificar todos.

2. **Atualizar a versão** — incrementar conforme convenção (X.0 / X.Y / X.Y.Z)

3. **Atualizar este CHANGELOG** — adicionar entrada descrevendo o que mudou e por quê

4. **Atualizar a tabela "Estado atual"** no topo deste arquivo

5. **Comunicar Felipe se for impactar agentes em produção** — alguns documentos alimentam diretamente prompts dos agentes do Squad Hub. Mudança neles pode exigir re-deploy.
